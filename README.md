# ARP and ICMP Communication Analysis in a LAN

## Overview
This project demonstrates basic network communication within a Local Area Network (LAN) using Cisco Packet Tracer.

The topology consists of two laptops and one desktop PC connected through a switch. Static IP addresses were assigned to all devices to enable communication within the same subnet.

## Objectives
- Configure a simple LAN environment
- Verify connectivity between hosts
- Analyze ARP requests and replies
- Observe ICMP Echo Request and Echo Reply packets generated by the ping command

## Network Configuration

| Device | IP Address |
|----------|------------|
| Laptop 1 | 192.168.1.1 |
| Laptop 2 | 192.168.1.2 |
| PC | 192.168.1.3 |

## Activities Performed
1. Created the network topology in Cisco Packet Tracer.
2. Assigned static IPv4 addresses to each device.
3. Tested connectivity using the `ping` command.
4. Switched to Simulation Mode.
5. Captured and analyzed:
   - ARP Request
   - ARP Reply
   - ICMP Echo Request
   - ICMP Echo Reply

## Key Concepts
- LAN Communication
- ARP (Address Resolution Protocol)
- ICMP
- IPv4 Addressing
- Network Connectivity Testing

## Outcome
The exercise successfully demonstrated how devices discover MAC addresses through ARP and exchange ICMP packets to verify connectivity within the same LAN.
